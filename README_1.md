

# 📘 Інструкція з керування репозиторієм (Workflow)

Цей проект налаштований на роботу з двома репозиторіями:

1. **origin** — мій особистий репозиторій для збереження роботи.

2. **teacher** — репозиторій викладача для отримання оновлень.


### 🔍 1. Перевірка статусу (Де я зараз?)

Перед будь-якою дією перевір, де ти знаходишся:

- `git remote -v` — **Найважливіша перевірка.** Має показати:

  - `origin` -\> `Lexx23021074/my-js-course-work` (мій)

  - `teacher` -\> `GoITteacher/fs-128-js` (викладача)

- `git branch` — покаже поточну гілку (зазвичай `main`).

- `git status` — чи є незакомічені зміни.


### 📥 2. Як отримати оновлення від викладача

Якщо викладач вніс зміни в оригінальний репозиторій:

1. **Заховати свої поточні напрацювання** (щоб не було конфліктів при затягуванні):

Bash

```
***`git stash`**
```

2. **Оновити інформацію про гілки викладача:**

Bash

```
***`git fetch teacher`**
```

3. **Затягнути зміни з гілки main викладача у свою гілку:**

Bash

```
***`git pull teacher main`**
```

4. **Повернути свої зміни назад:**

Bash

```
***`git stash pop`**
```

*(Якщо з'явилися конфлікти — Git підсвітить файли червоним. Виправ їх у VS Code, збережи і зроби коміт).*


### 📤 3. Як зберегти свою роботу у свій GitHub

Коли ти закінчив працювати і хочеш "залити" код собі:

1. **Додати всі файли:**

Bash

```
***`git add .`**
```

2. **Зафіксувати зміни:**

Bash

```
***`git commit -m "Опис того, що зробив (наприклад: виконав дз 2)"`**
```

3. **Відправити у СВІЙ репозиторій:**

Bash

```
***`git push origin main`**
```


### 🛠 4. Короткий довідник команд

| **Команда** | **Що робить** |
| - | - |
| `git remote -v` | Перевірити адреси обох репозиторіїв |
| `git pull teacher main` | Взяти нове у викладача |
| `git push origin main` | Віддати своє на мій GitHub |
| `git stash` | Тимчасово "заморозити" незавершену роботу |
| `git stash pop` | Розморозити роботу назад |
